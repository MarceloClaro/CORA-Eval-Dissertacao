\section{Instrumento de Avaliacao (TDD Pedagogico)}

Cada exercicio do modulo e um \textbf{teste} que verifica um criterio
de aprendizado. Se o aluno passa no teste, ha evidencia de que aprendeu.
Se falha (RED), o modulo fornece \textbf{feedback corretivo} (GREEN).

\begin{center}
\begin{longtable}{p{0.06\textwidth}p{0.28\textwidth}p{0.22\textwidth}p{0.28\textwidth}}

\textbf{CT} & \textbf{Exercicio} & \textbf{Criterio de Aprovacao} & \textbf{Feedback se Falhar} \\
\midrule

\section{Ciclo de Evolucao do Raciocinio Multiagente}

\subsection{O Que o Aluno Observa ao Usar o OpenCode}

Quando o aluno executa um experimento no OpenCode, ele nao esta apenas
rodando codigo \^a?? esta \textbf{participando de um ciclo de evolucao de
raciocinio}. O sistema demonstra, em tempo real, como o conhecimento
cientifico avanca:

\begin{enumerate}[nosep]
 \item \textbf{Multiagente:} 79 agentes especializados trabalham em
 paralelo \^a?? o aluno ve \textit{quem} fez \textit{o que}.
 \item \textbf{Multiconhecimento:} Cada agente contribui com uma
 perspectiva diferente (matematica, estatistica, computacional,
 logica).
 \item \textbf{Verificacao cruzada:} O Cora-Debate audita cada
 afirmacao \^a?? o aluno ve \textit{por que} algo foi aprovado
 ou rejeitado.
 \item \textbf{Evolucao:} O AutoEvolve registracao aprendizado e
 sugere o proximo passo \^a?? o aluno ve o \textit{progresso}
 acumulado.
\end{enumerate}

\subsection{Checklist de Reproducao}

\begin{enumerate}[nosep]
 \item[$\square$] Python 3.12+ instalado (\texttt{python --version})
 \item[$\square$] Git instalado (\texttt{git --version})
 \item[$\square$] Repositorio clonado (\texttt{git clone ...})
 \item[$\square$] Dependencias instaladas (\texttt{pip install -r requirements.txt})
 \item[$\square$] Teste de sanidade passa (\texttt{python -c "from core.container import Container"})
 \item[$\square$] TDD executado (\texttt{pytest test\_gat\_curvatura.py -v})

 \item[$\square$] Artigo gerado (\texttt{opencode run /artigo})
